8 - PUZZLE

Background :

The 8 puzzle also often called the 9-puzzle is a 3x3 tiles puzzle where the objective is to place the tiles in order by making sliding moves that use the empty space. The twist in this app is that the challenge is to do it in as few moves as possible. There is a minimum amount of moves you have to make (the optimal solution) that sets how difficult it is to solve the puzzle. If you manage to move the same amount of moves as the optimal solution you get the score of the difficulty. Each move above the difficulty level will be subtracted from the score. The scoreboard has room for top five scores. If you get a new higscore you will be prompted to write your name or initials and save it.


How it works:

The app is a one screen app. It has one slider for difficulty. Moving this slider will update the difficulty text box (pass information from one activity to the next). When you then hit the SHUFFLE button (button which triggers an action when clicked), the app will shuffle the labels and background color on the 9 buttons that makes out the puzzle according to the number of moves shown by the difficulty number. For this to work, an algorithm giving an optimal solution has to be designed for the shuffling. There are several to choose from.

When you push a number button (button which triggers an action when clicked) close to the blank button (dark grey here in wireframe below), the button and blank will exchange background color and labels. This will be counted as a move and the "moves" text box will be incremented showing one more move. If it is to hard, you can push the SOLVE button (button which triggers an action when clicked) and the app will show the moves with a 1 second delay, slow enough to see the solution.

When you solve the puzzle the background image that has been invisible will be brought to the front covering the puzzle buttons (display an image). If you make it to the top five list, you will get a popup message (display a pop-up message ("toast")) where you can write in your name or initials and save it to disk (save information on disk and use it in future executions).

The SCOREBOARD button shows a popup message (display a pop-up message ("toast")) with a list of top five results.

Wireframe:

Layout

The layout is sketched in the wireframe sketch above. The application is a one screen application. The application has a horizontal layout of 6 rows. 5 of the rows takes up 1/10 of the space of the screen making up for half the screen toghether. The puzzle takes up the other half the screen. Some of the rows has nested horizonal layout and/or vertical layout (use at least 2 layouts). The layout is explained in the list below.

Horizontal Layout:

  1. First row is a header (taking up 1/10 of screen)
  2. Second row is a difficulty slider (taking up 1/10 of screen)
  3. Third row (taking up 1/10 of screen horizontally) has a vertical layout:
    1. First column has a text saying "moves" (taking up 1/2 of screen)
    2. Second column has a textbox stating current number of moves (taking up 1/2 screen)
  4. Fourth row (taking up 1/10 of screen horizontally) has a vertical layout:
    1. First column has a text saying "difficulty" (taking up 1/2 of screen)
    2. Second column has a textbox stating the (difficulty) minimum number of moves (taking up 1/2 screen)
  5. The fifth row is divided into a horizontal layout of 3 rows and a each row has a vertical layout of 3 columns. This row has a background image taking up the whole row size which is 1/2 of the screen size. the 3 rows are evenly space making up 1/3 of the horizontal layout row. The 3 columns is evenly spaced 1/3 of screen size.
  6. Last row (taking up 1/10 of screen horizontally) is a button row with vertical layout of 3 columns with 3 buttons.
    1. First column has a shuffle button (taking up 1/4 of screen)
    2. Second column has a scoreboard button (taking up 1/2 screen)
    3. Third column has a solve button (taking up 1/4 of screen)
